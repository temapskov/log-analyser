# ADR-0003: Формат отчёта — Markdown по умолчанию

- **Статус:** Accepted
- **Дата:** 2026-04-23
- **Автор:** architect
- **Связанные ADR:** [ADR-0004](0004-templating-text-template.md) — шаблонизатор.

## Контекст

По FR-4 для каждого из 5 хостов формируется отдельный файл `<host>_YYYY-MM-DD.<ext>`. OQ-4 не зафиксировал формат; аналитик предложил `.md` по умолчанию, оставив развилку архитектору.

Целевые свойства файла:
- **Читаемость в TG на мобильном.** Telegram показывает превью `.md` с подсветкой заголовков и списков.
- **Diff-friendly** для ручной сверки / bug-reports.
- **Плотность информации** — не тратим рендеринг на UI-мишуру.
- **Не ломается** в web-viewer'е (GitHub, GitLab).

## Решение

**`.md` (CommonMark)** как формат по умолчанию. Опция `REPORT_FORMAT=md|html|txt` в конфиге — для будущих форматов. В MVP реализуем только `md`.

Структура файла — см. `docs/plans/00-analysis.md §15.1`. Основные секции: заголовок, сводка, агрегаты по app, топ-N инцидентов, паттерны, шум.

Таблицы — через GitHub-flavored `| col | col |`. Блоки кода для выдержек `_msg`. Ссылки на Grafana — inline `[текст](url)`.

## Последствия

**Плюсы**
- Человеко-читаемо без рендера (plain text годится).
- Telegram отображает `.md` с частичной разметкой в preview — пользователь видит структуру без скачивания.
- Универсальный формат: можно открыть в любом редакторе, VS Code, GitHub gist, браузерном viewer'е.
- Сравнение «что поменялось день ко дню» диффабельно (`diff --git a/t5.md b/t5.md`).
- Шаблон `.tmpl` → `.md` строго plain-text, без html-escaping-сюрпризов.

**Минусы / trade-offs**
- Нет интерактивных элементов (сортировка таблиц) — для этого есть UI v0.3.
- Длинные URL Grafana в inline-ссылках читаемы плохо — маскируем текстом `[Grafana →]`, сам URL — внутри.
- При очень большом числе инцидентов файл тяжёлый для восприятия — компенсируется `TOP_N=20` + блок «прочее».

## Альтернативы

1. **`.html`.** Отвергнуто на MVP: требует `html/template` + CSS, жирнее, TG preview всё равно показывает только часть. Оставлено как опция `REPORT_FORMAT=html` на будущее (FR-15 UI уже `html/template`, можно переиспользовать).
2. **`.txt` plain.** Отвергнуто: теряются визуальные ориентиры — заголовки, таблицы. Плохо читается.
3. **`.gz`-архив с `.log` внутри.** Отвергнуто: пользователь не будет распаковывать ради утреннего чтения. Рассматривается как последний шаг при превышении 50 МБ (R-3).
4. **PDF.** Отвергнуто: безумный overhead (движок рендеринга), CGO (`gofpdf` — ок, но headless-chrome — нет), нечитаемо без viewer'а.
5. **JSON (структурированно).** Отвергнуто как user-facing: для UI v0.3 или external consumers можем экспортировать отдельно, но daily-report — для человека.
